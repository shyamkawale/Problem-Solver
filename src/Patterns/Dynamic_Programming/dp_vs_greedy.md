2. The Million Dollar Question: Is it DP or Greedy?
This is the hardest part of algorithm interviews. Here is a mental framework to help you decide.

The "Future Regret" Test (The Golden Rule)
Ask yourself: "If I make the best choice right now, could I regret it later?"

GREEDY (No Regret):

In the string problem: If you see a Type A and Type B and swapping is cheap, you do it. Will you ever say, "Oh no, I wish I hadn't swapped those, because now I can't solve the last 2 characters"? No. Fixing an error is always good. The errors don't interact with each other in complex ways.

Key Signal: Local optimal choice = Global optimal solution.

DP (Fear of Regret):

Example: 0/1 Knapsack Problem. You see a heavy Gold Bar. It's valuable! A greedy person takes it. But taking it fills your bag, so you can't fit the 3 Diamonds you find later. You regret taking the gold.

Key Signal: Your choices consume a shared resource (capacity, time, stamina) that limits your future options.

Intuition: Why separate "Type A" and "Type B"?The core reason we count Type A (s=0, t=1) and Type B (s=1, t=0) separately is how the Swap operation works.Think of it like Trading Cards:Type A: You have a 0, but you want a 1.Type B: You have a 1, but you want a 0.Scenario 1: You have two Type AsYou: "I have a 0, I need a 1."Friend: "I also have a 0, I also need a 1."Action: If you swap your 0 with his 0, nothing changes. You both still have the wrong card.Conclusion: Swapping identical errors is useless.Scenario 2: You have one Type A and one Type BYou: "I have a 0, I need a 1."Friend: "I have a 1, I need a 0."Action: If you swap, you get the 1 you needed, and he gets the 0 he needed. One move fixes both problems.Conclusion: This is the "Magic Pair." This is why we separate the counts—we are desperately looking for these pairs because they are usually the most efficient fix.The Intuition:The problem isn't just "how many errors are there?" It is "how many errors can simply cancel each other out?" Type A and Type B are inverses. Separating them allows you to see how many "cancellations" (swaps) are possible versus how many "brute force" fixes (flips/crosses) are required for the leftovers.2.
